## Who This Track Is For

- Engineers new to OpenShift or containers
- Linux admins transitioning into Kubernetes
- Developers who want to understand how OpenShift actually works
- Anyone preparing for DO180 or moving toward EX280

---

## Goals

By completing this track, you will:

- Understand core OpenShift and Kubernetes concepts
- Interact confidently with OpenShift using the `oc` CLI
- Deploy and troubleshoot containerized applications
- Learn how OpenShift builds on Kubernetes primitives
- Develop good verification and debugging habits early

---

## Philosophy

These labs are meant to be:

- **Hands-on first**, not slide-driven
- **Typed and practiced**, not blindly copy/pasted
- **Verified continuously**, not assumed correct
- **Concept-aware**, explaining *why* when it matters
- **EX280-aligned**, building skills that carry forward

This is not an answer key or shortcut repo.
